# MBB Consulting Document Standards

Quality bar for all consulting deliverables. Every document — Word or PowerPoint — must meet these standards. Standards draw from McKinsey, Bain, and BCG best practices.

## Core Principles

### 1. The Pyramid Principle (Barbara Minto)

Lead with the answer. Group 2-4 MECE supporting arguments. Support each with evidence.

```
          [Answer / Recommendation]
         /          |              \
   [Reason 1]  [Reason 2]    [Reason 3]
    /    \       /    \         /    \
 [Data] [Data] [Data] [Data] [Data] [Data]
```

**Never** build up to the conclusion. Lead with the "so what."

### 2. MECE Thinking

All frameworks, categorizations, and analyses must be:
- **Mutually Exclusive**: No overlapping categories
- **Collectively Exhaustive**: Nothing material left out

### 3. Situation-Complication-Resolution (SCR)

Every document opens with:
- **Situation**: Current context (1-2 sentences, uncontroversial)
- **Complication**: What changed or what's at stake
- **Resolution**: The recommendation

### 4. Action Titles, Not Topic Titles

Every heading must be a complete sentence stating the key message.

| Bad (Topic Title) | Good (Action Title) |
|---|---|
| Competitive Landscape | Five competitors are in Phase 3, with survodutide posing the greatest differentiation risk |
| Market Overview | The MASH market will exceed $25B by 2030, driven by GLP-1 RA approvals |
| Recommendations | Prioritize BTD application and initiate Phase 3 design by Q3 to maintain competitive positioning |

A reader should understand the full argument from titles alone.

### 5. One Message Per Unit

- One message per slide (PowerPoint)
- One message per paragraph (Word)
- One message per exhibit
- If a slide needs two messages, it's two slides

## PowerPoint Standards

### Structure
```
Slide 1:  Title Slide (project, client, date, confidentiality)
Slide 2:  Executive Summary (findings + recommendation)
Slide 3:  Agenda / Document Map
Slide 4+: Content slides (one message per slide)
Penultimate: Recommendations / Next Steps
Final:    Appendix divider → supporting data
```

### Slide Layout
```
┌─────────────────────────────────────────────────┐
│ ACTION TITLE: Complete sentence stating the       │
│ key message of this slide                         │
├─────────────────────────────────────────────────┤
│                                                   │
│         EVIDENCE / EXHIBIT                        │
│   (chart, table, framework, structured text)      │
│                                                   │
├─────────────────────────────────────────────────┤
│ Source: [Specific citation]        │ Footnotes    │
└─────────────────────────────────────────────────┘
```

### Slide Rules
1. Action title: Bold, 18-22pt, sentence case
2. One exhibit per slide
3. Source line on every data slide (specific citation)
4. Consistent formatting throughout
5. No decorative elements (no clip art, stock photos, gradients, animations)
6. Ghost deck first: build outline with action titles before content
7. Maximum 25 content slides (move overflow to appendix)
8. Page numbers on every slide except title

### Executive Summary Slide
Must contain:
- Context: 1 sentence
- Key findings: 3-5 bullets, each starting with bolded action phrase
- Recommendation: 1-2 sentences
- Next steps: 2-3 actions with owners and timing

### Chart Standards
- Title = takeaway ("Revenue will grow 3x by 2030" not "Revenue Forecast")
- Label axes with units
- Callout boxes for key data points
- Max 5-7 data series
- Source line required

### Table Standards
- Action-oriented column headers
- Highlight key row/column
- Consistent number formatting
- Zebra striping for >5 rows
- Bold summary/total row

### Color Palette
- **Primary**: Dark navy (#1B2A4A) — titles and text
- **Accent 1**: Medium blue (#2D6BB4) — emphasis, primary data
- **Accent 2**: Teal (#1A8A7D) — secondary data
- **Accent 3**: Warm gray (#8C8C8C) — supporting elements
- **Alert**: Deep red (#C23B22) — risks/critical items only
- **Background**: White or light gray (#F5F5F5)
- Max 4 colors per slide

## Word Document Standards

### Structure
```
Cover Page (title, client, date, confidentiality, version)
Table of Contents
Executive Summary (1-2 pages maximum)
1. Introduction / Background
   1.1 Situation
   1.2 Complication / Objectives
2-N. Body sections (each opens with SCR or action-titled subsections)
N+1. Recommendations
N+2. Next Steps
Appendix A: Methodology
Appendix B: Detailed Data
Appendix C: References
```

### Formatting Rules
1. Font: Calibri or Arial, 11pt body, 14pt headings, 16pt section titles
2. Line spacing: 1.15 body, 1.0 tables
3. Margins: 1 inch all sides
4. Headers/footers: Document title in header, page numbers in footer, confidentiality
5. Paragraph spacing: 6pt after each paragraph
6. Maximum 2 levels of bullets

### Writing Rules
1. Active voice: "We recommend..." not "It is recommended..."
2. Short sentences: 15-20 words target
3. Short paragraphs: 3-5 sentences, one idea each
4. Bold key phrases: First 3-5 words of key paragraphs
5. Exhibits embedded in text (not in appendix, except detailed supporting data)
6. Cross-references: "As shown in Exhibit 3" not "as shown below"

### Executive Summary Rules
- Maximum 2 pages
- Must stand alone
- Structure: Situation → Key Findings (3-5 numbered) → Recommendation → Next Steps
- No new information in the body that isn't foreshadowed here

### Exhibit Standards
Every exhibit must have:
1. Exhibit number: "Exhibit 1:", "Exhibit 2:", etc.
2. Action title: "Exhibit 1: Survodutide poses the greatest competitive threat based on Phase 3 data"
3. Source line: Below, left-aligned, smaller font
4. In-text reference

## Consulting Deliverable Templates

### Due Diligence Report (Word)
```
Cover Page — [Asset] Due Diligence | [Client] | [Date] | CONFIDENTIAL
Executive Summary (2 pages max)
  Overall Assessment: Go / Conditional Go / No-Go
  Key Findings (5-7 numbered)
  Top 3 Risks
  Recommendation with timeline
1. Asset Overview (target, MOA, IP)
2. Clinical Assessment (efficacy, safety, development timeline)
3. Regulatory Assessment (pathway, precedents, designations)
4. Competitive Landscape (approved, pipeline, positioning)
5. Commercial Assessment (market sizing, revenue model, pricing)
6. Risk-Adjusted Valuation (PTS, rNPV, comparable transactions)
7. Overall Assessment and Recommendation (scorecard, go/no-go, next steps)
Appendices
```

### Due Diligence Deck (PowerPoint, 12 slides)
```
1. Title
2. Executive Summary — Go/No-Go with 3 key reasons
3. Asset overview — mechanism, indication, differentiation
4. Clinical data — key efficacy + safety
5. Development timeline — milestones vs. competitors
6. Regulatory pathway — precedent, designations
7. Competitive landscape — N competitors, differentiation
8. Differentiation matrix
9. Market opportunity — $[X]B market, peak revenue
10. Risk assessment — 3 key risks + mitigations
11. Valuation — rNPV base/upside/downside
12. Recommendation — Go/No-Go + next steps
```

### Market Access Strategy (Word)
```
Cover Page — Market Access Strategy | [Product] | CONFIDENTIAL
Executive Summary (2 pages)
1. Unmet Need and Disease Burden
2. Product Value Proposition
3. Payer Landscape Analysis (US + EU HTA bodies)
4. Competitive Pricing Analysis
5. Health Economics (CEA, BIA)
6. Access Strategy (formulary positioning, restrictions, contracting)
7. Evidence Generation Roadmap
8. Launch Sequencing Strategy
9. Recommendations and Next Steps
Appendices
```

### Market Access Deck (PowerPoint, 12 slides)
```
1. Title
2. Exec summary — [Product] can achieve [access] at [$X-Y] if [gaps] addressed
3. Unmet need
4. Value proposition vs. comparator
5. Payer landscape — key decision-makers, evidence requirements
6. Pricing analysis — benchmarks, recommended corridor
7. Evidence gaps — 3 priorities
8. HTA readiness
9. Access barriers + mitigation
10. Launch sequencing
11. Evidence roadmap — studies, timeline, investment
12. Recommendations — price, studies, payer engagement
```

### Client Proposal (Word)
```
Cover Page — [Project Title] | Proposal for [Client] | CONFIDENTIAL
1. Executive Summary (SCR, 1 page)
2. Understanding of the Challenge (2-3 pages)
3. Our Approach (3-4 pages, workstream structure)
4. Timeline and Milestones
5. Team
6. Investment
7. Why [Your Firm]
Appendices: Bios, Case Studies, Terms
```

### Proposal Deck (PowerPoint, 12 slides)
```
1. Title
2. We understand your challenge
3. The key questions
4. Our approach — framework overview
5-7. Workstream 1-3 detail
8. Timeline with milestones
9. Your team + credentials
10. Why us — differentiators
11. Investment — fee + deliverables
12. Next steps
```
