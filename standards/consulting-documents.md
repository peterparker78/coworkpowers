# MBB Consulting Document Standards

This reference defines the quality bar for all consulting deliverables. Every document — Word or PowerPoint — must meet these standards before delivery. All Work agents producing consulting deliverables should apply these principles.

These standards draw from McKinsey, Bain, and BCG best practices for professional services documents.

## Core Principles

### 1. The Pyramid Principle (Barbara Minto)

Every document and every slide follows the pyramid:
- **Lead with the answer.** The recommendation or key message comes first, not last.
- **Group supporting arguments.** Support the answer with 2-4 mutually exclusive, collectively exhaustive (MECE) reasons.
- **Support each argument with evidence.** Data, analysis, and citations underneath each argument.

```
          [Answer / Recommendation]
         /          |              \
   [Reason 1]  [Reason 2]    [Reason 3]
    /    \       /    \         /    \
 [Data] [Data] [Data] [Data] [Data] [Data]
```

**Never** build up to the conclusion. Consultants lead with the "so what."

### 2. MECE Thinking

All frameworks, categorizations, and analyses must be:
- **Mutually Exclusive**: No overlapping categories
- **Collectively Exhaustive**: Nothing material left out
- If a framework isn't MECE, restructure it before writing.

### 3. Situation-Complication-Resolution (SCR)

Every document or major section opens with:
- **Situation**: The current context (1-2 sentences, uncontroversial)
- **Complication**: What changed or what's at stake (the tension that demands action)
- **Resolution**: The recommendation or answer (the rest of the document supports this)

### 4. Action Titles, Not Topic Titles

Every heading, every slide title, every section header must be an **action title** — a complete sentence that states the key message, not just the topic.

| Bad (Topic Title) | Good (Action Title) |
|---|---|
| Competitive Landscape | Five competitors are in Phase 3, with survodutide posing the greatest differentiation risk |
| Market Overview | The MASH market will exceed $25B by 2030, driven by GLP-1 RA approvals |
| Regulatory Considerations | FDA's accelerated approval pathway is validated but requires a 7-year confirmatory trial commitment |
| Recommendations | Prioritize BTD application and initiate Phase 3 design by Q3 to maintain competitive positioning |

A reader should be able to read ONLY the titles and understand the full argument.

### 5. One Message Per Unit

- **One message per slide** (PowerPoint)
- **One message per paragraph** (Word)
- **One message per exhibit** (tables and figures)
- If a slide needs two messages, it's two slides.

## PowerPoint Standards

### Structure

```
Slide 1:  Title Slide (project name, client, date, confidentiality)
Slide 2:  Executive Summary (key findings + recommendation in 1 slide)
Slide 3:  Agenda / Document Map
Slide 4+: Content slides (one message per slide)
Penultimate: Recommendations / Next Steps
Final:    Appendix divider (then supporting data slides)
```

### Slide Layout

```
┌─────────────────────────────────────────────────┐
│ ACTION TITLE: Complete sentence stating the       │
│ key message of this slide                         │
├─────────────────────────────────────────────────┤
│                                                   │
│         EVIDENCE / EXHIBIT                        │
│   (chart, table, framework, or structured text)   │
│                                                   │
│                                                   │
│                                                   │
├─────────────────────────────────────────────────┤
│ Source: [Specific citation]        │ Footnotes    │
└─────────────────────────────────────────────────┘
```

### Slide Rules

1. **Action title**: Bold, 18-22pt, top of slide. Complete sentence. No title case — sentence case.
2. **One exhibit per slide**: Don't crowd. If it needs two charts, make two slides.
3. **Source line on every data slide**: Bottom left. Specific citation (not "various sources").
4. **Consistent formatting**: Same font family, same color palette, same table formatting throughout.
5. **No decorative elements**: No clip art, no stock photos, no gradients, no animations.
6. **Ghost deck first**: Build the full outline with action titles before adding any content. If the titles tell a coherent story, the deck is well-structured.
7. **Maximum 25 content slides** for a standard engagement deliverable. If longer, restructure or move content to appendix.
8. **Page numbers on every slide** except the title slide.

### Executive Summary Slide

The single most important slide. Must contain:
- **Context**: 1 sentence
- **Key findings**: 3-5 bullets, each starting with a bolded action phrase
- **Recommendation**: 1-2 sentences, clearly labeled
- **Next steps**: 2-3 specific actions with owners and timing

A senior partner should be able to read this slide alone and make a decision.

### Chart and Table Standards

**Charts:**
- Title is the takeaway, not the metric name ("Revenue will grow 3x by 2030" not "Revenue Forecast")
- Label axes clearly with units
- Use callout boxes to highlight the key data point
- Limit to 5-7 data series maximum
- Source line required

**Tables:**
- Column headers must be action-oriented where possible
- Highlight the most important row or column
- Use consistent number formatting (1 decimal, $ or %, always aligned)
- Zebra striping for readability on tables >5 rows
- Bold the summary/total row

### Color Palette

Use a professional, restrained palette:
- **Primary**: Dark navy (#1B2A4A) for titles and text
- **Accent 1**: Medium blue (#2D6BB4) for emphasis and primary data
- **Accent 2**: Teal (#1A8A7D) for secondary data
- **Accent 3**: Warm gray (#8C8C8C) for supporting elements
- **Alert**: Deep red (#C23B22) for risks or critical items only
- **Background**: White or very light gray (#F5F5F5)
- Never use more than 4 colors on a single slide.

## Word Document Standards

### Structure

```
Cover Page (title, client, date, confidentiality, version)
Table of Contents
Executive Summary (1-2 pages maximum)
1. Introduction / Background
   1.1 Situation
   1.2 Complication / Objectives
2-N. Body sections (each opens with SCR or action-titled subsections)
N+1. Recommendations
N+2. Next Steps
Appendix A: Methodology
Appendix B: Detailed Data
Appendix C: References / Bibliography
```

### Formatting Rules

1. **Font**: Calibri or Arial, 11pt body, 14pt headings, 16pt section titles
2. **Line spacing**: 1.15 for body text, 1.0 for tables
3. **Margins**: 1 inch all sides (2.54 cm)
4. **Headers/footers**: Document title in header, page numbers in footer, confidentiality marking
5. **Paragraph spacing**: 6pt after each paragraph
6. **Maximum 2 levels of bullets**: If you need 3 levels, restructure the content

### Writing Rules

1. **Active voice**: "We recommend..." not "It is recommended that..."
2. **Short sentences**: Target 15-20 words. Break complex sentences.
3. **Short paragraphs**: 3-5 sentences maximum. One idea per paragraph.
4. **Bold key phrases**: The first 3-5 words of each key paragraph should be bolded to enable scanning.
5. **Exhibits embedded in text**: Tables and figures go where they're discussed, not in an appendix (except for very detailed supporting data).
6. **Cross-references**: "As shown in Exhibit 3" not "as shown below" (page breaks change).

### Executive Summary Rules

The executive summary is the most important section. Rules:
- **Maximum 2 pages** regardless of document length
- **Must stand alone**: Someone reading only the executive summary should understand the situation, key findings, and recommendation
- **Structure**: Situation (2-3 sentences) → Key Findings (3-5 numbered points) → Recommendation (1-2 sentences) → Next Steps (bullet list)
- **No new information in the body that isn't foreshadowed in the executive summary**

### Exhibit Standards

Every exhibit (table, figure, framework) must have:
1. **Exhibit number**: "Exhibit 1:", "Exhibit 2:", etc.
2. **Action title**: "Exhibit 1: Survodutide poses the greatest competitive threat based on Phase 3 data and BTD designation"
3. **Source line**: Below the exhibit, left-aligned, smaller font
4. **In-text reference**: Every exhibit must be referenced in the body text

## Quality Checklist (Apply to Every Deliverable)

### Structure
- [ ] Pyramid principle: Answer first, then support
- [ ] SCR framing in introduction/opening
- [ ] Action titles on every heading/slide
- [ ] One message per slide/paragraph
- [ ] Executive summary stands alone
- [ ] MECE frameworks throughout

### Content
- [ ] Every claim has a cited source
- [ ] Data is current (note cutoff dates)
- [ ] Competitive data is comprehensive (not cherry-picked)
- [ ] Risks and limitations are acknowledged
- [ ] Recommendations are specific and actionable
- [ ] Next steps have owners and timelines

### Formatting
- [ ] Consistent font, size, and color throughout
- [ ] Page numbers on every page/slide
- [ ] Source lines on every data exhibit
- [ ] Confidentiality marking on every page
- [ ] Version number and date on cover
- [ ] Professional color palette (no bright primary colors)
- [ ] Tables and charts properly labeled

### Tone
- [ ] Authoritative but not arrogant
- [ ] Data-driven, not opinion-driven
- [ ] Client-first framing (their challenges, their opportunities)
- [ ] Active voice throughout
- [ ] No filler words, no padding
